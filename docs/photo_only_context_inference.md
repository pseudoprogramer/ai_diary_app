# 사진 기반 하루 맥락 추론

사용자는 캘린더에 모든 일정을 기록하지 않는다. 점심 약속, 자주 가는 카페, 산책, 친구와의 가벼운 만남처럼 실제 하루의 많은 순간은 캘린더에 남지 않는다. 따라서 하루결은 캘린더가 없어도 사진첩만으로 하루의 맥락을 어느 정도 복원할 수 있어야 한다.

## 원칙

- 사진 원본은 외부 API로 보내지 않는다.
- 사진 분석은 최대한 기기 안에서 처리한다.
- 서버나 LLM으로 보내는 정보는 원본 이미지가 아니라 로컬에서 추출한 요약 신호만 사용한다.
- 확신이 낮은 추론은 단정하지 않고 사용자에게 확인한다.

## 로컬에서 추출할 신호

### 메타데이터

- 촬영 시간
- GPS 위치
- 사진 묶음 간 시간 간격
- 같은 장소에서 반복 촬영 여부
- 이동 패턴

### 비전 분석

- 장면 분류: 음식, 카페, 실내, 야외, 강, 바다, 산, 꽃, 야경, 문서 등
- 객체 탐지: 컵, 접시, 음식, 책, 노트북, 사람, 꽃, 의자, 테이블 등
- 사람 존재 여부
- 얼굴 수는 사용할 수 있지만 얼굴 인식/식별은 기본적으로 하지 않는다.
- 사진 임베딩을 이용한 비슷한 사진 묶기

## 온디바이스 분석 후보

### iOS

- Apple Vision framework
- Core ML custom classifier/object detector
- 사람/동물/텍스트/분류/마스크 요청을 조합

### Android

- MediaPipe Image Classifier
- MediaPipe Image Embedder
- TensorFlow Lite Object Detection
- TensorFlow Lite custom classifier

## 사진만으로 가능한 추론 예시

### 점심 약속

입력 신호:

- 12:10 음식 사진 3장
- 12:35 카페 음료 사진 1장
- 같은 위치 반경 150m 안에서 1시간 이상 머묾
- 사람 손/테이블/접시 탐지

추론:

- `점심 약속 또는 외식`

### 카페 작업

입력 신호:

- 커피, 노트북, 책상, 실내 사진
- 14:00~17:00 같은 장소 체류
- 캘린더 일정 없음

추론:

- `카페에서 작업하거나 공부한 시간`

### 한강 산책

입력 신호:

- 강/하늘/꽃/길/노을 사진
- GPS가 한강공원 근처
- 18:00~19:30 사이 이동 흔적

추론:

- `한강 산책`

## 내부 구조

사진은 먼저 시간과 장소 기준으로 클러스터링한다.

```json
{
  "clusterId": "cluster_2026_04_30_1800",
  "start": "18:04",
  "end": "19:12",
  "placeHint": "여의도 한강공원",
  "photoCount": 8,
  "sceneLabels": ["river", "sunset", "flower", "walking path"],
  "objectLabels": ["food", "blanket", "cup"],
  "peoplePresent": true,
  "inferredActivity": "한강 피크닉과 산책",
  "confidence": 0.78
}
```

이 클러스터를 하루 타임라인의 기본 단위로 사용한다.

## 캘린더와의 관계

캘린더는 필수가 아니라 보조 신호다.

- 캘린더 있음: 일정과 사진 클러스터를 매칭해 정확도 향상
- 캘린더 없음: 사진 클러스터만으로 활동 후보 생성
- 캘린더와 사진이 충돌: 사용자에게 확인

예:

- 캘린더에는 `집` 일정만 있음
- 사진은 15시에 카페 사진 5장
- 앱 질문: `15시쯤 카페 사진이 여러 장 있어요. 카페에서 보낸 시간으로 기록할까요?`

## 프라이버시 정책

- 얼굴 식별은 하지 않는다.
- 사람 수나 인물 존재 여부 정도만 사용한다.
- 원본 사진은 로컬 저장소와 사진첩 안에서만 처리한다.
- LLM에 보낼 수 있는 정보는 다음 수준으로 제한한다.
  - `12시쯤 음식 사진 3장`
  - `15시쯤 카페/노트북 사진 2장`
  - `18시쯤 강/노을/꽃 사진 5장`
- 민감한 라벨은 사용자 확인 전 일기에 포함하지 않는다.

## MVP 우선순위

1. 오늘 사진 가져오기
2. EXIF 시간/GPS 추출
3. 시간 간격 기준 클러스터링
4. 위치가 가까운 사진끼리 묶기
5. 로컬 장면 분류 모델 붙이기
6. 규칙 기반 활동 추론
7. 낮은 확신도는 사용자 확인 질문으로 처리

## 1차 구현 전략

처음부터 큰 멀티모달 모델을 넣지 않는다.

- 1차: 시간/GPS/사진 개수만으로 클러스터 생성
- 2차: 로컬 이미지 분류 모델로 음식/카페/풍경/문서/사람 정도 라벨링
- 3차: 객체 탐지와 임베딩으로 묶음 품질 향상
- 4차: 고성능 기기에서 로컬 VLM 또는 더 큰 분류 모델 검토

이 구조라면 캘린더를 꼼꼼히 쓰지 않는 사용자도 사진첩만으로 하루 기록을 만들 수 있다.
